# 🌏 i18n guide

拟借助 [Weblate](https://hosted.weblate.org/engage/comicguispider/) 实现多语言的翻译，看后续收费与否而定  

Plan translate by [Weblate](https://hosted.weblate.org/engage/comicguispider/), see if it's paid or not later

## Development

### Ui

基于分支 `i18n` ，由 `Weblate` 翻译后的内容会对应所属语言的 `yaml` 文件如 [`en_US.yaml`](../../assets/res/locale/en-US.yml)，
然后经由 [`transfer.py`](../../assets/res/transfer.py) 编译成文件 [`en-US/LC_MESSAGES/res.mo`](../../assets/res/locale/en-US/LC_MESSAGES/res.mo)  

Based on the `i18n` branch, the content translated by `Weblate` will correspond to the `yaml` file of the language, such as [`en_US.yaml`](../../assets/res/locale/en-US.yml),
and then compiled into the file [`en-US/LC_MESSAGES/res.mo`](../../assets/res/locale/en-US/LC_MESSAGES/res.mo) by [`transfer.py`](../../assets/res/transfer.py)

### Documentation
文档皆存放在 `docs` 目录里，将来会通过 `github action` 做成 `github pages`  

documents are stored in the `docs` directory, which will be made into `github pages` in the future by `github action`

## Usage

软件在开启时，会通过 [`assets/res/__init__.py`](../../assets/res/__init__.py) 中的 `getUserLanguage` 函数获取当前机器的语言如 `Chinese (Simplified)_China.utf8`（`ISO 639-1`_`ISO 3166-1`.encoding），
检测转换成 `RFC1766` 标准如 `zh-CN` ，并加载 `*.mo` 模块进行语言切换。  
若进行了语言开发，需在 `getUserLanguage` 中增加对应的 `RFC1766` 转换，否则默认使用 `en-US`  

The software will get the current machine language such as `Chinese (Simplified)_China.utf8` (`ISO 639-1`_`ISO 3166-1`.encoding)
through the `getUserLanguage` function in [`assets/res/__init__.py`](../../assets/res/__init__.py) when it starts,
machine language will convert to the `RFC1766` standard such as `zh-CN`, and load the corresponding `*.mo` module for language switching.  
If language development has been carried out, you need to add the corresponding `RFC1766` conversion in `getUserLanguage`, otherwise the default `en-US` is used
